# ТЗ: главное меню игры «Дома Пустыни» — экран «Песчаный аванпост»

## 1. Назначение
Сверстать главный экран меню мобильной игры в стиле пустынного аванпоста. Основа — мобильный first-подход: минимум мелких элементов, крупные кликабельные зоны, акцент на 3 главных действия:

1. **В бой** — главный CTA.
2. **Коллекция карт** — второй по важности CTA.
3. **Выбор дома / фракции** — горизонтальный блок ниже.

Экран должен быть проще, чем ранняя версия: без лишних мелких меню, без перегруженной левой панели, без сундуков/заданий/пропусков.

---

## 2. Что должно быть на экране

### 2.1. Верхняя зона
#### Блок игрока (слева сверху)
Содержимое:
- герб/щит текущего дома;
- имя игрока: **Игрок**;
- текущий дом: **Дом Чести**;
- уровень в щите: **12**;
- прогресс-бар: **1250 / 2000**.

Клик:
- по всему блоку — открыть профиль игрока.

#### Блок ресурсов (справа сверху)
Содержимое:
- специи: **4250**;
- кристаллы: **560**.

Клик:
- по специям — открыть магазин специй;
- по кристаллам — открыть магазин кристаллов.

---

### 2.2. Заголовок экрана
По центру сверху:
- **ПЕСЧАНЫЙ АВАНПОСТ**
- ниже: **Уровень 1**
- ещё ниже — небольшой декоративный ромб/маркер.

Клик:
- открывает карту/экран локаций (пока можно заглушку).

---

### 2.3. Центральная hero-зона
Фон:
- пустыня;
- деревянно-каменный аванпост;
- красные знамена;
- башни/укрепления;
- тёплая песчаная атмосфера;
- справа — скальные массивы, на фоне небо и птицы.

Это **не просто картинка**, а фон главного меню. UI должен быть читаемым поверх него.

---

### 2.4. Главная кнопка
#### Кнопка «В БОЙ»
Главный элемент экрана.

Форма:
- крупный ромб;
- яркая золотая светящаяся окантовка;
- тёплый коричнево-янтарный центр;
- внутри герб;
- крупный текст: **В БОЙ**.

Клик:
- старт боя / переход на экран матча.

Состояния:
- default;
- hover / touch highlight;
- pressed;
- disabled (если понадобится позже).

Анимация:
- мягкое свечение контура;
- лёгкий pulse 1 раз в 2–3 секунды.

---

### 2.5. Вторичная главная кнопка
#### Кнопка «Коллекция карт»
Должна быть **заметной и крупной**, потому что пользователь явно хочет часто в неё заходить с телефона.

Форма:
- широкая горизонтальная кнопка;
- золотой контур;
- слева иконка карт;
- текст: **Коллекция карт**.

Расположение:
- сразу под кнопкой «В бой».

Клик:
- открывает экран коллекции карт.

Состояния:
- default;
- hover / touch;
- pressed.

---

### 2.6. Блок выбора фракции / дома
Расположение:
- ниже «Коллекции карт».

Состав:
- 4 карточки домов.

#### Карточка 1 — активная
- **Дом Чести**;
- яркая;
- в красно-оранжевой гамме;
- активная золотая окантовка;
- виден персонаж/силуэт;
- доступна и выбрана.

#### Карточки 2–4 — заблокированы
- **Дом Железа**;
- **Пустынные Кланы**;
- **Орден Голоса**.

Для них:
- затемнённая карточка;
- диагональный крест/перечёркивание;
- иконка замка по центру;
- подпись дома внизу;
- нельзя выбрать сейчас;
- по клику можно показывать сообщение: «Откроется на следующих уровнях».

---

### 2.7. Нижняя кнопка
#### Кнопка «Рейтинг»
Расположение:
- по центру внизу.

Форма:
- компактная горизонтальная капсула/плашка;
- иконка столбиков;
- текст: **Рейтинг**.

Клик:
- открыть экран рейтинга.

---

## 3. Что убрать
Из этого экрана убрать как отдельные элементы:
- сундуки;
- задания;
- пропуск;
- отдельные мелкие боковые кнопки;
- нижнее меню «Инженеры», «Магазин» и т.п.;
- лишние маленькие кликабельные элементы.

Экран должен быть сфокусирован на 4 сущностях:
1. Профиль игрока;
2. Ресурсы;
3. В бой;
4. Коллекция карт;
5. Блок домов/фракций;
6. Рейтинг.

---

## 4. Иерархия важности
1. **В бой** — самый заметный элемент.
2. **Коллекция карт** — второй по заметности.
3. **Название локации**.
4. **Активная карточка дома**.
5. **Профиль и ресурсы**.
6. **Рейтинг**.

---

## 5. Компоновка по экрану (mobile portrait first)
Базовый макет рассчитан на **1080×1920** как референс, но верстка должна быть адаптивной.

### Референсные пропорции
- Верхний левый блок игрока: x=3%, y=2%, w=49%, h=12%
- Верхний правый блок ресурсов: x=55%, y=3%, w=42%, h=8%
- Заголовок локации: x=14%, y=13%, w=72%, h=9%
- Кнопка «В бой»: x=28%, y=41%, w=44%, h=28%
- Кнопка «Коллекция карт»: x=14%, y=69%, w=72%, h=8%
- Карточки домов: x=5%, y=79%, w=90%, h=16%
- Кнопка «Рейтинг»: x=34%, y=95%, w=32%, h=4%

Важно: значения можно адаптировать, но визуальная иерархия должна сохраниться.

## 7. Responsive / адаптация

### 7.1. Mobile portrait (главный сценарий)
Разрешения:
- 360×800
- 390×844
- 414×896
- 430×932

Требования:
- все основные клики не меньше 44×44 px;
- лучше 52–64 px для важных кнопок;
- между кнопками достаточно воздуха;
- заголовок не должен ломаться в 3 строки.

### 7.2. Mobile landscape
- hero-кнопку уменьшить;
- карточки домов сделать компактнее;
- блоки игрока и ресурсов расположить одной строкой сверху;
- рейтинг можно сместить вправо вниз.

### 7.3. Tablet
- сохранить композицию;
- добавить больше воздуха вокруг центральной кнопки;
- не возвращать удалённые второстепенные элементы.

## 8. Визуальный стиль

### Палитра
Основные цвета:
- фон тёмный: `#0D0A07`
- тёплый песочный: `#B97B3D`
- золотой контур: `#F4B54A`
- яркий glow: `#FFCC66`
- тёплый коричневый центр кнопок: `#4C2610`
- тёмный бронзовый: `#2A170D`
- почти чёрный для плашек: `#12100D`
- красный акцент дома: `#8E2A12`
- тёмный серо-синий для lock-карт: `#27303A`
- тёмный фиолетово-серый для дальнейшей фракции: `#362D45`

### Стиль рамок
- тонкая золотая рамка;
- лёгкое свечение на активных элементах;
- округлые углы у панелей;
- ромб — только для CTA и декоративных маркеров.

### Шрифтовый характер
Нужно что-то с ощущением стратегии/песков/дома, ближе к serif display.
В коде допустимо:
- заголовки: `Cinzel`, `Cormorant Garamond`, `Playfair Display` или аналог;
- интерфейс/цифры: `Inter`, `Manrope`, `Merriweather Sans` или аналог.

---

## 9. Анимации
Минимум:
- fade-in экрана;
- лёгкое свечение battleButton;
- hover glow / active glow;
- небольшое движение пыли на фоне;
- плавное появление lock popup.

Не делать тяжёлые анимации, которые мешают кликабельности.

## 12. Технические требования к верстке
- mobile first;
- чистая компонентная структура;
- вынести цвета/радиусы/тени в tokens;
- отдельные SVG-элементы использовать как вставляемые компоненты;
- важно, чтобы CTA и карточки были легко масштабируемы.

---

## 13. Что входит в комплект файлов
Вместе с ТЗ отдать:
1. **SVG-файл с векторными UI-деталями**;
2. **HTML-файл с уже описанными кодом блоками**;
3. **данное ТЗ**.

Набор SVG должен содержать:
- щит игрока;
- верхнюю панель игрока;
- верхнюю панель ресурсов;
- ромб CTA;
- вторичную кнопку;
- карточку активной фракции;
- карточку заблокированной фракции;
- кнопку рейтинга;
- маленький декоративный ромб;
- lock badge.

---

## 14. Definition of Done
Готово, если:
- экран похож по структуре на референс;
- на мобильном всё читаемо;
- «В бой» и «Коллекция карт» самые заметные;
- заблокированные дома явно выглядят недоступными;
- нет лишнего перегруза мелкими пунктами меню;
- есть быстрый путь для внедрения через SVG или через готовые HTML/CSS блоки.
