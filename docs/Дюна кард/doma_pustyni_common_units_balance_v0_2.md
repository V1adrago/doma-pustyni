# Дома Пустыни — общие юниты и баланс башен v0.2

**Тип документа:** отдельный дизайн-документ по общим юнитам  
**Версия:** v0.2  
**Дата:** 05.06.2026  
**Назначение:** описать базовый пул общих юнитов без фракций, их боевую логику, силу между собой, баланс башен и параметры для реализации в коде.

---

## 1. Принцип базовой версии

В первой боевой версии используются только общие юниты. Фракционные юниты, уникальные способности домов и домовые карты не добавляются.

Цель этого этапа — проверить базовую боевую петлю:

1. Игрок тратит специи на юнита.
2. Юнит появляется на выбранной линии.
3. Юнит движется вперёд.
4. Если встречает врага — останавливается и дерётся.
5. Если врагов нет — идёт к башне.
6. Если дошёл до башни — атакует башню.
7. Башня стреляет по врагам в радиусе.
8. Если башня уничтожена — линия открывается к цитадели.

Ключевое правило: **контра не означает запрет боя**. Все наземные боевые юниты могут бить все наземные цели. Контра означает только более выгодный размен по специям, HP, DPS и времени.

---

## 2. Глобальные правила боя

### 2.1. Наземные юниты не проходят сквозь врага

Если наземный боевой юнит встречает вражеского наземного боевого юнита на своей линии, он останавливается и атакует его.

Исключения:

- инженер не атакует, но его можно атаковать;
- стрелок останавливается на дистанции атаки;
- воздушный юнит не блокируется наземными юнитами;
- башня считается зданием, а не юнитом.

### 2.2. Все наземные боевые юниты размениваются ударами

Мечник может бить тяжёлого воина. Разведчик может бить тяжёлого воина. Штурмовик может бить тяжёлого воина. Но выгодность размена будет разная.

Пример:

- один мечник задерживает тяжёлого и снимает ему часть HP;
- два мечника уже могут остановить тяжёлого при поддержке башни;
- копейщик является лучшим экономическим ответом на тяжёлого, но не единственным способом его убить.

### 2.3. Воздушные юниты

Воздушный юнит не блокируется наземными юнитами и не получает от них урон, если у наземного юнита нет атаки по воздуху.

В базовой версии воздух атакуют:

- стрелок;
- боковая башня;
- цитадель.

Воздух не атакуют:

- разведчик;
- мечник;
- штурмовик;
- копейщик;
- тяжёлый воин.

### 2.4. Башни участвуют в бою

Башня — это не только экономический объект. Башня должна стрелять.

Боковые башни защищают фланги. Цитадель сильнее боковой башни. Башни атакуют землю и воздух.

## 5. Юниты подробно

### 5.1. Инженер

**Роль:** экономический юнит.  
**Стоимость:** Инженер I — 2 специи, Инженер II — 3 специи, Инженер III — 4 специи.  
**Задача:** дойти до центрального ресурсного узла.

Инженер не атакует юнитов и не атакует башни. Он идёт по центральной линии к ресурсному узлу. Если дошёл — активирует или улучшает инженерную добычу специй и исчезает. Если погиб до узла — стадия добычи не активируется.

| Параметр | Значение |
|---|---:|
| HP | 140 |
| Скорость | 1.00 |
| Урон | 0 |
| Дальность | нет |
| Цели | нет |

**Как использовать:** выпускать в центр и прикрывать боевым юнитом.  
**Как контрить:** разведчиком, стрелком, быстрым давлением по центру.  
**Важное правило:** инженерный старт должен быть рискованным, потому что реальная цена выхода — инженер плюс защитник.

### 5.2. Разведчик

**Роль:** быстрый перехватчик.  
**Стоимость:** 2 специи.  
**Задача:** убивать инженера, давить пустую линию, добегать до стрелка.

Разведчик — самый быстрый наземный юнит. Он не должен быть сильным в лобовом бою. Его сила — скорость и ранний темп.

| Параметр | Значение |
|---|---:|
| HP | 220 |
| Скорость | 1.60 |
| Урон по юнитам | 32 |
| Урон по башням | 18 |
| Кулдаун атаки | 0.8 сек |
| DPS по юнитам | 40 |
| Дальность | 0.8 |

**Силен против:** инженер, стрелок без прикрытия.  
**Слаб против:** мечник, башня, тяжёлый воин.  
**Боевой смысл:** разведчик быстро убивает инженера, если игрок не прикрыл экономический выход.

### 5.3. Мечник

**Роль:** базовый защитник фланга.  
**Стоимость:** 2 специи.  
**Задача:** держать линию, защищать инженера, останавливать разведчика.

Мечник — простой универсальный солдат. Он не обязан выигрывать все дуэли, но обязан вступать в бой и задерживать угрозу.

| Параметр | Значение |
|---|---:|
| HP | 430 |
| Скорость | 0.95 |
| Урон по юнитам | 45 |
| Урон по башням | 32 |
| Кулдаун атаки | 1.0 сек |
| DPS по юнитам | 45 |
| Дальность | 0.9 |

**Силен против:** разведчик, слабые ближние юниты, стрелок если добежал.  
**Слаб против:** воздушный юнит, тяжёлый воин 1 на 1, штурмовик при прямом пуше.  
**Боевой смысл:** один мечник задерживает тяжёлого, но не обязан его убивать. Два мечника или мечник под башней уже могут остановить тяжёлую цель.

### 5.4. Штурмовик

**Роль:** базовый атакующий по башням.  
**Стоимость:** 3 специи.  
**Задача:** дойти до башни и нанести заметный урон.

Штурмовик не должен проходить сквозь защиту. Если его встречает наземный юнит, он останавливается и дерётся. Его преимущество раскрывается, если он доходит до здания.

| Параметр | Значение |
|---|---:|
| HP | 680 |
| Скорость | 0.75 |
| Урон по юнитам | 42 |
| Урон по башням | 95 |
| Кулдаун атаки | 1.5 сек |
| DPS по башням | 79 |
| Дальность | 0.9 |

**Силен против:** башни, медленная слабая защита.  
**Слаб против:** копейщик, стрелок за передней линией, башня без поддержки.  
**Боевой смысл:** первый юнит, который реально заставляет защищать башню.

### 5.5. Стрелок

**Роль:** дальняя поддержка и базовый анти-воздух.  
**Стоимость:** 3 специи.  
**Задача:** бить из-за спины, защищать от дрона, снимать HP с медленных юнитов.

Стрелок должен быть слабым вблизи. Он силён только при правильной позиции за мечником, штурмовиком или тяжёлым воином.

| Параметр | Значение |
|---|---:|
| HP | 190 |
| Скорость | 0.85 |
| Урон по земле | 38 |
| Урон по воздуху | 50 |
| Урон по башням | 18 |
| Кулдаун атаки | 1.0 сек |
| Дальность | 4.5 |

**Силен против:** дрон, медленные юниты на дистанции, копейщик.  
**Слаб против:** разведчик, мечник если добежал, башня.  
**Боевой смысл:** без стрелка игрок уязвим к воздуху.

### 5.6. Копейщик

**Роль:** анти-тяжёлый защитник.  
**Стоимость:** 3 специи.  
**Задача:** выгодно размениваться против тяжёлого воина и штурмовика.

Копейщик бьёт всех наземных юнитов. Против обычных целей его урон средний. Против heavy и assault он наносит повышенный урон.

| Параметр | Значение |
|---|---:|
| HP | 320 |
| Скорость | 0.85 |
| Базовый урон | 35 |
| Урон по heavy / assault | 65 |
| Урон по башням | 18 |
| Кулдаун атаки | 0.9 сек |
| Дальность | 1.4 |

**Силен против:** штурмовик, бронированный воин.  
**Слаб против:** стрелок, дрон, атаки по двум линиям.  
**Боевой смысл:** не единственный ответ на тяжёлых, а самый выгодный ответ по экономике.

### 5.7. Песчаный дрон

**Роль:** первый воздушный юнит.  
**Стоимость:** 4 специи.  
**Задача:** наказывать игрока без анти-воздуха.

Дрон летит над линией и не блокируется наземными ближними юнитами. Он атакует землю и башни, но не атакует воздух.

| Параметр | Значение |
|---|---:|
| HP | 280 |
| Скорость | 1.10 |
| Урон по юнитам | 32 |
| Урон по башням | 30 |
| Кулдаун атаки | 1.0 сек |
| Дальность | 2.8 |
| Тип | air |

**Силен против:** мечник, копейщик, штурмовик, инженер.  
**Слаб против:** стрелок, башня, цитадель.  
**Боевой смысл:** заставляет иметь в колоде ответ на воздух, но не должен слишком быстро ломать башни.

### 5.8. Бронированный воин

**Роль:** тяжёлый танк.  
**Стоимость:** 5 специй.  
**Задача:** идти впереди, принимать урон, создавать медленный пуш.

Тяжёлый воин побеждает большинство дешёвых юнитов 1 на 1, но не проходит бесплатно. Каждый защитник задерживает его и снимает часть HP.

| Параметр | Значение |
|---|---:|
| HP | 900 |
| Скорость | 0.50 |
| Урон по юнитам | 60 |
| Урон по башням | 45 |
| Кулдаун атаки | 1.5 сек |
| DPS по юнитам | 40 |
| Дальность | 0.9 |

**Слаб против:** копейщик, дрон, фокус башни и нескольких юнитов.  
**Боевой смысл:** основа тяжёлого пуша, но требует поддержки и плохо реагирует на экономичный анти-тяжёлый ответ.

### 6.3. Приоритет башни

Для MVP башня выбирает цель просто:

1. Ищет ближайшего врага в радиусе.
2. Проверяет, может ли атаковать тип цели.
3. Стреляет по выбранной цели.
4. Если цель умерла или вышла из радиуса, выбирает новую.

Сложный приоритет по самому опасному юниту пока не нужен.

## 7. Сила между юнитами

Матрица показывает не возможность атаки, а выгодность размена.

Обозначения:

- **+** — выгодный размен;
- **0** — примерно равный размен;
- **−** — невыгодный размен;
- **не бьёт** — юнит физически не может атаковать эту цель.

| Атакующий / Цель | Разведчик | Мечник | Штурмовик | Стрелок | Копейщик | Дрон | Тяжёлый |
|---|---:|---:|---:|---:|---:|---:|---:|
| Разведчик | 0 | − | − | + если добежал | − | не бьёт | − |
| Мечник | + | 0 | − | + если добежал | 0 | не бьёт | − |
| Штурмовик | + | + | 0 | 0 если дошёл | − | не бьёт | − |
| Стрелок | + на дистанции | + на дистанции | + на дистанции | 0 | + | + | − |
| Копейщик | 0 | 0 | + | − | 0 | не бьёт | + |
| Дрон | + | + | + | − | + | 0 | + |
| Тяжёлый | + | + | + | + если дошёл | − | не бьёт | 0 |

## 8. Базовые боевые сценарии

### 8.1. Инженер без защиты против разведчика

Разведчик быстро добегает до инженера. Инженер не атакует. Разведчик наносит 30 урона каждые 0.8 секунды. Инженер с 140 HP умирает примерно за 4 секунды.

Вывод: инженерный старт без защиты должен проигрывать разведчику.

### 8.2. Инженер плюс мечник против разведчика

Мечник перехватывает разведчика. Разведчик может нанести урон инженеру, но в прямом бою проигрывает мечнику. Инженер получает шанс дойти до центрального узла.

Вывод: инженерный старт требует вложения в защиту.

### 8.3. Штурмовик против башни

Штурмовик медленно идёт к башне. Если его не встретили, он наносит 80 урона по башне раз в 1.2 секунды. Один штурмовик не должен ломать полную башню, но должен нанести заметный урон.

Вывод: штурмовик заставляет реагировать.

### 8.4. Тяжёлый воин против мечника

Тяжёлый воин побеждает одного мечника, но тратит время и теряет HP. Если мечник стоит под башней или их два, тяжёлый может быть остановлен.

Вывод: тяжёлый юнит не проходит сквозь лёгкую защиту бесплатно.

### 8.5. Дрон против игрока без стрелка

Мечник, копейщик, штурмовик и тяжёлый не могут атаковать воздух. Если у игрока нет стрелка, дрон получает безопасное давление.

Вывод: стрелок является базовым обязательным ответом на воздух.

## 9. Логика реализации в коде

```js

  range: 0.9,
  unitType: "ground",
  armorClass: "medium",

  canBlockGround: true
}
```

### 9.2. Типы юнитов

```js
unitType: "ground" | "air"
armorClass: "engineer" | "light" | "medium" | "assault" | "ranged" | "antiHeavy" | "heavy"
```

### 9.3. Типы целей

```js
targetTypes: ["ground", "air", "building"]
```

Примеры:

```js
// Стрелок
targetTypes: ["ground", "air", "building"]

// Мечник
targetTypes: ["ground", "building"]

// Инженер
targetTypes: []
```

### 9.4. Выбор цели

Для MVP:

1. Найти врагов в радиусе атаки или обнаружения.
2. Оставить только цели, которые юнит может атаковать.
3. Выбрать ближайшую цель.
4. Если цель есть — остановиться и атаковать.
5. Если цели нет — двигаться вперёд.
6. Если впереди башня в радиусе — атаковать башню.
7. Если башня уничтожена — двигаться к цитадели.

### 9.5. Бонусный урон копейщика

```js
function getDamage(attacker, target) {
  if (attacker.id === "spearman" && (target.armorClass === "heavy" || target.armorClass === "assault")) {
    return 65;
  }
  return attacker.attackDamage;
}
```

## 10. Итоговая логика MVP

Базовая боевая модель строится на простых и понятных ролях:

- инженер даёт экономику;
- разведчик мешает инженеру и наказывает слабую линию;
- мечник держит линию и защищает инженера;
- штурмовик давит башню;
- стрелок даёт дальний бой и ответ на воздух;
- копейщик выгодно режет тяжёлых и штурмовых;
- дрон наказывает отсутствие анти-воздуха;
- бронированный воин танкует и собирает медленный пуш;
- башни стреляют и являются активной частью обороны.

Главная формула боя:

**юниты не проходят сквозь друг друга; все наземные дерутся со всеми наземными; воздух требует отдельного ответа; башни стреляют; контра — это выгодный размен, а не запрет боя.**

## 11. Доработка баланса башен и юнитов v0.2

### 11.1. Главная проблема текущего ощущения

В текущей версии юниты ощущаются слишком слабыми относительно башни. Если башня слишком легко убивает одиночных и средних юнитов, игрок перестаёт бояться оставлять линию без защиты. Тогда бой превращается в постоянный пуш других направлений, а не в борьбу за контроль линий.

Правильная философия:

**Башня — это усилитель обороны, а не полноценная армия.**

Башня должна помогать защитнику, добивать слабых юнитов и наказывать одиночные ошибки. Но если игрок не защищает линию юнитами, атакующая волна должна быть страшной силой.

### 11.2. Запрещённая логика: урон от смерти юнита

Башня не должна получать дополнительный урон из-за того, что она убила юнита.

Правило:

**Башня получает урон только от атак юнита, пока юнит жив и находится в радиусе атаки по зданию.**

Запрещено:

- урон башне после смерти юнита;
- компенсационный урон за стоимость карты;
- скрытый урон за то, что юнит дошёл до башни;
- взрыв после смерти, если это не отдельная явно описанная карта.

Кодовая фиксация:

```js
// Правильно: урон только от атаки живого юнита
if (unit.isAlive && unit.target === tower && unit.canAttack(tower)) {
  tower.hp -= unit.buildingDamage;
}

// После смерти юнит больше не наносит урон
if (!unit.isAlive) {
  unit.stopAttacking();
}
```

Не добавлять:

```js
// Так делать нельзя
tower.hp -= unit.deathDamage;
tower.hp -= unit.remainingValueDamage;
tower.hp -= unit.costCompensationDamage;
```

### 11.3. Почему юниты слабеют против башни

Проблема не только в уроне башни. Главная проблема — связка из трёх параметров:

1. большой радиус башни;
2. высокий DPS башни;
3. малое время, за которое юнит успевает атаковать башню.

Если боковая башня имеет радиус 6.5, а ближний штурмовик атакует с дистанции 0.9, он получает урон почти весь путь до башни. При скорости 0.75 он идёт под огнём примерно 7.5 секунд до первой атаки. За это время башня успевает сильно просадить HP юнита ещё до того, как он начал наносить урон по зданию.

Значит, базовая правка — не добавлять искусственный урон, а дать юниту больше времени жить и реально бить башню.

### 11.4. Новая роль башни

Башня одна должна справляться с мелкой угрозой, но не должна бесплатно держать полноценную волну.

| Ситуация | Нормальный результат |
|---|---|
| Один разведчик дошёл до башни | башня убивает, получает малый урон |
| Один мечник дошёл до башни | башня убивает, но теряет заметный HP |
| Один штурмовик дошёл до башни | башня получает серьёзный урон |
| Штурмовик + мечник дошли до башни | башня почти падает или теряет большую часть HP |
| Два штурмовика дошли до башни | боковая башня должна упасть без защиты |
| Башня + защитник против равной атаки | оборона должна иметь шанс выстоять |

Ключевая формула:

**Башня без защиты проигрывает сильной атаке. Башня с защитником должна держать равную по стоимости атаку.**

| Тест | Нормальный результат |
|---|---|

### 11.10. Итоговая формула баланса

Боевая модель должна прийти к такому состоянию:

**Один дешёвый юнит не ломает башню.**

**Один штурмовик без ответа должен быть страшен.**

**Волна за 5–6 специй должна заставлять защищаться.**

**Волна за 7–8 специй должна ломать боковую башню, если игрок не поставил защиту.**

**Башня не должна наносить скрытый урон сама себе через смерть юнитов. Весь урон башне идёт только от атак живых юнитов.**
