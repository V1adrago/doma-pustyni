
# Дома Пустыни — юниты и постановка на поле v0.9

Тип документа: отдельный файл по юнитам, drag-and-drop постановке и развороту целей  
Версия: v0.9  
Дата: 06.06.2026  
Назначение: зафиксировать простую механику постановки юнитов на поле, правила для текущего пула юнитов и боевую логику разворота.

---

## 1. Главная идея

Юнитов нужно ставить не только кликом по панели. Игрок должен перетаскивать карту из руки на поле и выбирать точку постановки.

Базовое правило простое:

**Обычные боевые юниты ставятся на своей половине поля до середины. Чужая половина закрыта. Если враг зашёл на твою половину, можно поставить юнита ему за спину. Враг должен потратить время на разворот и только потом атаковать.**

Это создаёт защиту через позиционирование.

---

## 2. Деление поля

| Зона | Правило |
|---|---|
| Своя половина | основная зона постановки |
| Середина поля | крайняя точка обычной постановки |
| Чужая половина | обычных юнитов ставить нельзя |

Ставить около башен соперника нельзя. Это должно быть редкой механикой будущих специальных карт, а не базовым правилом.

---

## 3. Два типа постановки

Для MVP достаточно двух типов.

| Тип | Описание | Кто использует |
|---|---|---|
| `own_half` | можно ставить на своей половине до середины | обычная наземная пехота |
| `backline` | можно ставить только из тыла / от башенной линии | инженер, осада, воздух |

Не нужно делать сложные проценты глубины для каждого юнита.

---

## 4. Правила по текущим юнитам

| cardId | Название | Тип постановки | Почему |
|---|---|---|---|
| `engineer` | Инженер Узла | `backline` | должен пройти путь до центрального узла |
| `scout` | Ищейка Барханов | `own_half` | быстрый перехват, слабый в лобовом бою |
| `swordsman` | Клинок Дома | `own_half` | базовая пехота, защита и добор волны |
| `assault` | Башнелом | `backline` | высокий урон по башням, нельзя давать старт у центра |
| `archer` | Песчаный Стрелок | `own_half` | пехотная поддержка, уязвим при близком контакте |
| `spearman` | Пикейщик Каравана | `own_half` | ответ на тяжёлых и осадных |
| `drone` | Дюнный Сокол | `backline` | воздух не блокируется землёй |
| `heavy` | Латник Пустыни | `own_half` | якорь пуша и сбор критической массы |
| `guard` | Гвардеец Чести | `own_half` | защитная пехота, не ломает башни быстро |

## 5. Почему тяжёлый юнит может ставиться до середины

Латник Пустыни тяжёлый, но его слабость не только в длинном пути от базы.

Его реальные слабости:

- высокая стоимость;
- низкая скорость;
- медленный разворот;
- зависимость от поддержки;
- уязвимость к Пикейщику Каравана;
- уязвимость к Дюнному Соколу;
- плохая реакция на давление по другой линии.

Поэтому его можно ставить на своей половине до середины. Он становится не мгновенной угрозой башне, а **якорем пуша**.

Сценарий:

1. Игрок ставит Латника ближе к середине своей половины.
2. Противник видит направление давления.
3. Игрок от тыла выпускает более быстрых юнитов.
4. Быстрые догоняют Латника.
5. Формируется критическая масса.
6. Если соперник не отвечает, башня получает сильное давление.

---

## 6. Постановка за спину врага

Постановка за спину врага разрешена, если точка находится на своей половине поля.

Это важно:

- вражеский юнит зашёл на твою половину;
- ты ставишь юнита позади него;
- враг должен заметить цель;
- враг должен остановиться;
- враг должен развернуться;
- только после этого он может атаковать.

Так дешёвые юниты получают ценность даже против тяжёлых.

---

## 7. Разворот как часть боевой логики

Разворот должен быть не только анимацией.

Правильная формула:

**цель сзади -> остановка -> разворот -> задержка -> атака**

Неправильная формула:

**цель сзади -> мгновенная атака с визуальным разворотом**

## 10. Агро и выбор цели

Юнит должен искать цели вокруг себя, а не только перед собой.

Базовая логика:

1. Если рядом есть атакуемый враг, выбрать его.
2. Если цель сбоку или сзади, войти в разворот.
3. Во время разворота не атаковать.
4. После turnDelay начать атаку.
5. Если врагов рядом нет, двигаться к башне.
6. Если башня в радиусе и врагов рядом нет, атаковать башню.
7. Если при атаке башни рядом появился враг, переключиться на него.

## 11. Drag-and-drop UX

Поток игрока:

1. Зажал карту в руке.
2. Потянул на поле.
3. Игра показала разрешённую зону.
4. Отпустил карту.
5. Если точка разрешена — юнит появился, специи списались.
6. Если точка запрещена — карта вернулась в руку, специи не списались.

Подсветка:

| Состояние | Цвет |
|---|---|
| Разрешённая зона | песочно-золотой |
| Запрещённая зона | красный |
| Ghost юнита | цвет нормальный или красный при запрете |

---

## 12. Онлайн

Событие постановки должно передавать не только карту и линию, но и точку постановки.

Было:

```js
{ cardId, lane }
```

Стало:

```js
{ cardId, lane, deployPoint: { x, z } }
```

Или:

```js
{ cardId, x, z }
```

Разворот не нужно синхронизировать отдельными событиями. Он должен детерминированно считаться на клиентах.

## 14. Итоговая формула

**Своя половина — зона тактической постановки.  
Пехота может ставиться до середины.  
Инженер, осада и воздух стартуют из тыла.  
Ставить за спину врагу можно, если это твоя половина.  
Враг обязан тратить время на разворот перед атакой.**
