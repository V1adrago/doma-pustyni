# Готовый промпт для Claude: drag-and-drop постановка и разворот юнитов


```text
Проанализируй текущий проект «Дома Пустыни» по README.md, docs/CLAUDE.md и текущему коду.

Нужно реализовать механику постановки юнитов через drag-and-drop и механику разворота юнитов при атаке цели сбоку/сзади.

Главная идея:
Сейчас юнитов можно выбрать через панель и поставить по простой логике линии. Нужно добавить возможность перетаскивать карту из руки на поле и выбирать точку постановки на своей половине поля.

ВАЖНО:
Не надо делать сложную систему процентов глубины для каждого юнита.
Базовое правило простое:
- обычные боевые юниты ставятся на своей половине поля до середины;
- чужая половина закрыта;
- ставить около башен соперника нельзя;
- если враг зашёл на твою половину, можно поставить юнита ему за спину;
- враг должен потратить время на разворот и только потом атаковать.

Файлы, которые нужно проверить и доработать:
- src/cards.js
- src/main.js
- src/unit-manager.js
- src/units.js
- src/hand.js, если нужно
- style.css
- index.html, если нужно добавить overlay подсветки
- src/services/network-service.js
- server/index.js, если онлайн-действие play_card сейчас не передаёт координату постановки
- README.md
- memory/project_doma_pustyni.md

ЗАДАЧА 1. Добавить правила постановки в CARD_DEFS

В src/cards.js каждому юниту добавить deployRules.

Использовать два типа постановки:

1. own_half
Обычная постановка. Юнита можно ставить на своей половине поля до середины.

2. backline
Тыловая постановка. Юнита можно ставить только в задней части своей половины / от башенной линии.

Распределение:

engineer:
deployRules: {
  placementType: 'backline',
  allowedLanes: ['center'],
  deployDelay: 0,
  canPlaceBehindEnemies: true
}

scout:
deployRules: {
  placementType: 'own_half',
  allowedLanes: ['left', 'center', 'right'],
  deployDelay: 0,
  canPlaceBehindEnemies: true
}

swordsman:
deployRules: {
  placementType: 'own_half',
  allowedLanes: ['left', 'center', 'right'],
  deployDelay: 0,
  canPlaceBehindEnemies: true
}

assault:
deployRules: {
  placementType: 'backline',
  allowedLanes: ['left', 'center', 'right'],
  deployDelay: 0.4,
  canPlaceBehindEnemies: true
}

archer:
deployRules: {
  placementType: 'own_half',
  allowedLanes: ['left', 'center', 'right'],
  deployDelay: 0,
  canPlaceBehindEnemies: true
}

spearman:
deployRules: {
  placementType: 'own_half',
  allowedLanes: ['left', 'center', 'right'],
  deployDelay: 0,
  canPlaceBehindEnemies: true
}

drone:
deployRules: {
  placementType: 'backline',
  allowedLanes: ['left', 'center', 'right'],
  deployDelay: 0.5,
  canPlaceBehindEnemies: true
}

heavy:
deployRules: {
  placementType: 'own_half',
  allowedLanes: ['left', 'center', 'right'],
  deployDelay: 0,
  canPlaceBehindEnemies: true
}

guard:
deployRules: {
  placementType: 'own_half',
  allowedLanes: ['left', 'center', 'right'],
  deployDelay: 0,
  canPlaceBehindEnemies: true
}

Не менять cardId, стоимость, HP, урон, скорость, дальность и кулдауны.

ЗАДАЧА 2. Добавить drag-and-drop из руки на поле

Нужно, чтобы карту можно было:
1. зажать в руке;
2. перетащить на поле;
3. увидеть подсветку разрешённой зоны;
4. отпустить;
5. если точка разрешена — списать специи и создать юнита;
6. если точка запрещена — вернуть карту в руку, специи не списывать.

Оставить старую механику клика как fallback, если drag-and-drop не сработал.

Для fallback:
- клик по карте может ставить юнита по старой логике на выбранную линию или в дефолтную точку своей половины.

ЗАДАЧА 3. Валидация зоны постановки

Нужно добавить функцию проверки:

isDeployPointAllowed(cardId, side, x, z)

Правила:

1. Для placementType = 'own_half':
- точка должна быть на своей половине поля;
- точка не должна быть дальше середины поля;
- точка может быть перед врагом, сбоку или за спиной врага, если это своя половина.

2. Для placementType = 'backline':
- точка должна быть на своей половине;
- точка должна быть в задней части своей половины / около своей башенной линии;
- точка не должна быть у середины;
- engineer можно ставить только на центральную линию.

3. Для всех обычных карт запрещено:
- ставить на чужой половине;
- ставить около башен соперника;
- ставить в чужую базу;
- ставить прямо на центральный узел, если это ломает инженера.

ЗАДАЧА 4. Привязка к линиям

После выбора точки юнит должен привязываться к ближайшей линии.

Линии:
- left: x = -6
- center: x = 0
- right: x = 6

Если игрок отпустил карту между линиями, выбрать ближайшую.

Передавать в spawn не только lane, но и точку постановки:

spawn(cardId, side, lane, deployPoint)

Если сложно менять сигнатуру, можно добавить необязательный параметр:

spawn(cardId, side, lane, options = { deployPoint: { x, z } })

Если deployPoint не передан, использовать старую дефолтную позицию.

ЗАДАЧА 5. Подсветка зоны постановки

При перетаскивании карты:
- разрешённая зона подсвечивается песочно-золотым цветом;
- запрещённая точка подсвечивается красным;
- можно показывать ghost-превью юнита на ближайшей линии.

Для MVP достаточно простой плоскости/оверлея:
- own_half — подсветить свою половину до середины;
- backline — подсветить заднюю часть своей половины;
- если курсор на запрещённой зоне, ghost сделать красным.

ЗАДАЧА 6. Разрешить постановку за спину врага на своей половине

Нельзя запрещать постановку рядом с врагом только потому, что это «за спину».

Правило:
Если точка находится на своей половине и подходит deployRules карты, постановка разрешена.

Это нужно для защитной тактики:
- враг зашёл на твою половину;
- ты ставишь юнита позади него;
- враг разворачивается;
- враг теряет время;
- башня продолжает стрелять.

ЗАДАЧА 7. Добавить turnDelay юнитам

В CARD_DEFS добавить turnDelay.

Рекомендуемые значения:

engineer: null
scout: 0.15
swordsman: 0.25
spearman: 0.30
archer: 0.35
guard: 0.40
assault: 0.45
heavy: 0.70
drone: 0.20

Разворот должен быть частью боевой логики, а не только визуальной анимацией.

ЗАДАЧА 8. Реализовать состояние разворота в unit-manager.js / Unit

Если юнит выбирает цель, которая находится сбоку или сзади относительно текущего направления движения/атаки, он должен:
1. остановиться;
2. войти в состояние turning;
3. накопить turnTimer;
4. не атаковать во время разворота;
5. после turnDelay начать атаку.

Правильная формула:
цель сзади -> остановка -> разворот -> задержка -> атака

Неправильно:
цель сзади -> мгновенная атака с визуальным разворотом

ЗАДАЧА 9. Агро вокруг юнита

Юнит должен искать врагов не только перед собой, а вокруг себя в радиусе агро.

Базовая логика:
1. Если рядом есть вражеский юнит, которого можно атаковать, выбрать его.
2. Если цель сбоку или сзади — развернуться с задержкой.
3. Если рядом нет юнитов — идти к башне.
4. Если рядом башня и нет юнитов — атаковать башню.
5. Если юнит атакует башню, но рядом появился вражеский юнит, который подходит по targetTypes, переключиться на него.

ЗАДАЧА 10. Онлайн

В онлайн-режиме play_card должен передавать точку постановки.

Было:
{ cardId, lane }

Стало:
{ cardId, lane, deployPoint: { x, z } }

Или:
{ cardId, x, z }

Второй клиент должен создать юнита в той же точке.

Не нужно синхронизировать поворот каждый кадр. Поворот должен считаться локально одинаково на обоих клиентах.

ЗАДАЧА 11. AI / бот

Если бот пока не умеет drag-and-drop, оставить ему простую логику:
- для own_half юнитов ставить ближе к середине своей половины;
- для backline юнитов ставить из тыла/от башенной линии;
- при защите можно ставить юнита за спину атакующему, если атакующий уже на половине бота.

ЗАДАЧА 12. Не менять баланс цифр

Не менять:

- стоимость;
- урон;
- урон по башням;
- скорость;
- дальность;
- cooldown;
- экономику специй;

Добавляем только:
```
